# Chrome Web Store 发布指南

本指南将帮助您将 CryptoPanel 发布到 Chrome Web Store。

## 📋 发布前准备

### 1. 更新版本号

在 `manifest.json` 中更新版本号：

```json
{
  "version": "1.0.0"  // 使用语义化版本号
}
```

**版本号规则**：
- `MAJOR.MINOR.PATCH`
- MAJOR：不兼容的 API 修改
- MINOR：向下兼容的功能性新增
- PATCH：向下兼容的问题修正

### 2. 构建生产版本

```bash
# 确保所有依赖已安装
npm install

# 构建扩展
npm run build:extension

# 检查 dist 目录
ls -la dist/
```

**确保 dist 目录包含**：
- `manifest.json`
- `background.js`
- `popup.html` 和 `popup.js`
- `options.html` 和 `options.js`
- `icon16.png`, `icon48.png`, `icon128.png`
- `assets/` 目录（CSS 和 JS 文件）

### 3. 准备图标

**必需尺寸**：
- 16x16 像素（工具栏图标）
- 48x48 像素（扩展管理页面）
- 128x128 像素（Chrome Web Store）

**图标要求**：
- PNG 格式
- 透明背景（可选，但推荐）
- 清晰可见，在小尺寸下也能识别
- 符合 Chrome Web Store 设计规范

**检查清单**：
- [ ] 所有尺寸图标都已准备
- [ ] 图标在 `public/` 目录中
- [ ] 构建后图标在 `dist/` 目录中

### 4. 准备截图

**必需截图**（至少 1 张，最多 5 张）：
- 最小尺寸：1280x800 或 640x400
- 推荐尺寸：1280x800 或 1920x1200
- 格式：PNG 或 JPEG

**截图建议**：
1. 主仪表板（显示资产列表）
2. 设置页面（显示配置选项）
3. 资产详情（如果有）
4. 多交易所支持展示
5. 主题切换展示

**截图工具**：
- Chrome 扩展：GoFullPage、Awesome Screenshot
- 或使用浏览器开发者工具截图

## 📝 创建必需文档

### 隐私政策模板

创建文件：`PRIVACY_POLICY.md`

```markdown
# CryptoPanel 隐私政策

**最后更新日期**：2024年[日期]

## 数据收集

CryptoPanel 是一个本地运行的 Chrome 扩展，我们**不会收集、存储或传输任何用户数据到外部服务器**。

### 本地存储的数据

以下数据仅存储在您的浏览器本地：

1. **交易所 API 配置**
   - API Key
   - Secret Key
   - Passphrase（如适用）
   - 这些信息仅用于访问您授权的交易所账户

2. **钱包地址**
   - 您添加的加密货币钱包地址
   - 仅用于查询链上资产

3. **应用设置**
   - 语言偏好
   - 主题设置
   - 显示偏好

4. **缓存数据**
   - 资产数据缓存（24小时）
   - 价格数据缓存（30分钟）

### 数据传输

扩展会向以下服务发送请求以获取数据：

1. **交易所 API**（仅当您配置了 API 密钥时）
   - Binance API：获取账户余额
   - OKX API：获取账户余额

2. **价格 API**（公开 API，无需认证）
   - DeFiLlama API：获取代币价格
   - CryptoCompare API：获取代币价格

3. **区块链 RPC 节点**（公开节点）
   - 用于查询链上资产余额

**重要**：所有 API 密钥和敏感信息仅存储在您的浏览器本地，不会传输到我们的服务器或任何第三方服务器。

### 数据安全

- 所有数据使用 Chrome 的 `chrome.storage.local` API 存储
- 数据仅存在于您的设备上
- 我们无法访问您的数据
- 卸载扩展将删除所有本地数据

### 第三方服务

我们使用以下第三方服务：

- **DeFiLlama**：价格数据（公开 API）
- **CryptoCompare**：价格数据（公开 API）
- **RPC 节点提供商**：链上数据查询（ThirdWeb、Infura、Alchemy）

这些服务可能有自己的隐私政策，请查看其官方网站。

### 您的权利

- 您可以随时查看、修改或删除存储的数据
- 卸载扩展将永久删除所有本地数据
- 我们不收集任何个人身份信息

### 联系我们

如有隐私相关问题，请通过以下方式联系：

- GitHub Issues：[项目地址]
- Email：[您的邮箱]

### 政策更新

我们可能会更新此隐私政策。重大更改将在扩展中通知用户。
```

## 🚀 发布步骤

### 步骤 1：注册 Chrome Web Store 开发者账号

1. 访问 [Chrome Web Store Developer Dashboard](https://chrome.google.com/webstore/devconsole)
2. 使用 Google 账号登录
3. 支付一次性注册费用：**$5 USD**（一次性，永久有效）
4. 完成开发者信息填写

### 步骤 2：打包扩展

1. 进入 `dist` 目录
2. 创建 ZIP 文件（**不要包含 dist 文件夹本身**）

```bash
cd dist
zip -r ../cryptopanel-v1.0.0.zip .
# 或使用其他工具创建 ZIP
```

**重要**：
- ZIP 文件应直接包含 `manifest.json`、`background.js` 等文件
- 不要包含 `dist` 文件夹
- 确保包含所有必需文件

### 步骤 3：上传到 Chrome Web Store

1. 登录 [Chrome Web Store Developer Dashboard](https://chrome.google.com/webstore/devconsole)
2. 点击"新项目"（New Item）
3. 上传 ZIP 文件
4. 填写扩展信息：

#### 基本信息
- **名称**：CryptoPanel
- **简短描述**：（见上方模板）
- **详细描述**：（见上方模板）
- **类别**：生产力工具（Productivity）
- **语言**：中文（简体）、英语

#### 图形资源
- **图标**：上传 128x128 PNG
- **截图**：上传至少 1 张截图（最多 5 张）
- **促销图片**（可选）：用于 Chrome Web Store 首页展示

#### 分类和网站
- **类别**：生产力
- **网站**（可选）：项目 GitHub 地址
- **支持网站**（可选）：GitHub Issues 地址

#### 隐私
- **单一用途**：是（推荐）
- **权限说明**：详细说明每个权限的用途
- **隐私政策 URL**：上传到 GitHub Pages 或提供可访问的 URL

**权限说明示例**：
```
storage: 用于在本地存储您的交易所配置、钱包地址和应用设置，所有数据仅存储在您的设备上。

alarms: 用于定期更新资产数据，您可以在设置中配置更新间隔。

host_permissions: 
- api.binance.com: 用于获取 Binance 交易所资产数据（仅当您配置了 Binance API 密钥时）
- www.okx.com: 用于获取 OKX 交易所资产数据（仅当您配置了 OKX API 密钥时）
- coins.llama.fi: 用于获取代币价格数据（公开 API）
- min-api.cryptocompare.com: 用于获取代币价格数据（公开 API）
- RPC 节点: 用于查询链上资产余额
```

#### 发布选项
- **可见性**：
  - **公开**：所有人可见（推荐用于正式发布）
  - **不公开**：仅通过链接访问（用于测试）
  - **受信任的测试人员**：仅测试人员可见
- **地区**：选择发布地区（或全球）

### 步骤 4：提交审核

1. 检查所有信息是否正确
2. 点击"提交审核"（Submit for Review）
3. 等待审核（通常 1-3 个工作日）

**审核检查清单**：
- [ ] 所有必填字段已填写
- [ ] 截图清晰且符合要求
- [ ] 隐私政策已提供
- [ ] 权限说明清晰
- [ ] 描述准确反映功能
- [ ] 版本号已更新
- [ ] 扩展功能正常

### 步骤 5：审核后

**如果审核通过**：
- 扩展将自动发布
- 您会收到邮件通知
- 可以在 Dashboard 查看下载统计

**如果审核被拒**：
- 查看拒绝原因
- 修复问题
- 重新提交

**常见拒绝原因**：
- 权限说明不清晰
- 隐私政策缺失或不完整
- 截图不符合要求
- 描述包含误导性信息
- 功能不符合 Chrome Web Store 政策

## 📊 发布后维护

1. 更新 `manifest.json` 中的版本号
2. 重新构建：`npm run build:extension`
3. 创建新的 ZIP 文件
4. 在 Developer Dashboard 中上传新版本
5. 填写更新说明（Release Notes）
6. 提交审核

### 监控和维护

- **用户反馈**：定期查看用户评论和反馈
- **错误监控**：监控扩展错误（如果集成了错误监控）
- **性能优化**：根据用户反馈优化性能
- **功能更新**：定期添加新功能和改进

### 版本管理建议

使用语义化版本号：
- `1.0.0` - 初始发布
- `1.0.1` - 修复 bug
- `1.1.0` - 新功能

## 🔍 审核要点

Chrome Web Store 审核重点关注：

1. **功能完整性**：扩展是否按描述工作

3. **权限合理性**：每个权限是否必要
4. **用户体验**：界面是否友好
5. **内容合规**：是否符合政策
